# 内容管理说明

这个项目已经把日常维护的内容目录改成中文了。你以后主要看这些地方：

## 最常改的内容

```text
src/content/文章        博客文章，Markdown 文件
src/content/日记        短日记，JSON 文件
src/content/相册        相册数据和相册图片
src/content/项目        项目展示
src/content/技能        技能卡片
src/content/时间线      时间线经历
src/content/友链        友情链接
src/content/关于.md     关于页面正文
src/content/友链.md     友链页面正文
```

## 文章怎么写

文章放在：

```text
src/content/文章
```

可以直接新建 `.md` 文件，也可以建中文子文件夹，例如：

```text
src/content/文章/随笔/第一篇文章.md
```

文章顶部需要 frontmatter，例如：

```md
---
title: "第一篇文章"
published: 2026-06-25
description: "这里是一句话摘要"
category: ["随笔"]
tags: ["生活", "记录"]
draft: false
---

这里开始写正文。
```

如果 `draft: true`，文章会被当作草稿，不会显示在正式网站上。

## 图片放哪里

公共背景、头像等图片：

```text
public/assets/images
```

文章配图建议直接放在文章同一个文件夹里，例如：

```text
src/content/文章/随笔/第一篇文章.md
src/content/文章/随笔/封面.jpg
```

然后在文章 frontmatter 里写：

```md
cover: "./封面.jpg"
```

正文里引用：

```md
![图片说明](./封面.jpg)
```

## 音乐播放器怎么加歌

音乐播放器配置在：

```text
twilight.config.yaml
```

现在已经设置成：

```yaml
musicPlayer:
    mode: "local"
    autoplay: false
```

也就是默认使用本地歌单，并且打开网站时不会自动播放。

音乐文件建议放在：

```text
public/assets/music
```

然后在 `twilight.config.yaml` 的 `musicPlayer.local.playlist` 里继续追加：

```yaml
            -
                id: 2
                title: "歌曲标题"
                artist: "歌手"
                cover: "/assets/images/avatar.jpg"
                url: "assets/music/歌曲文件名.mp3"
                lrc: "assets/music/歌词文件名.lrc"
                duration: 180
```

注意：

- `id` 每首歌要不一样。
- `url` 指向音乐文件。
- `lrc` 是歌词文件，没有歌词可以留空：`lrc: ""`。
- `duration` 是秒数，不准也没关系，播放器加载后会自动读取实际时长。

如果你不想把音乐文件放进 GitHub，也可以把 `url` 写成外部直链。但外部直链可能会失效，最稳的是本地放一份。

## 私密图片怎么放

如果只是挡普通访客，可以做一个“密码图片页”。但 GitHub Pages 是静态网站，不能保证真正保密。真正隐私、证件、私密原图不要放进网站仓库。

适合放网站里的情况：

```text
普通图片 / 半私密相册 / 不怕被技术用户找到的内容
```

不适合放网站里的情况：

```text
证件 / 隐私原图 / 账号密码 / 真实密钥 / 不能公开的资料
```

## AES 密码文章 / 私密相册怎么写

这个主题支持 AES 加密文章内容。现在已经改成可以从环境变量读取密码，这样密码不用写进公开仓库。

先在文章里这样写：

```md
---
title: "私密相册"
published: 2026-06-26
description: "需要密码查看"
category: ["私密"]
tags: ["相册"]
encrypted: true
passwordEnv: "PRIVATE_GALLERY_PASSWORD"
draft: false
---

# 私密相册

![图片说明](./photo-1.jpg)
![图片说明](./photo-2.jpg)
```

图片可以和文章放在同一个文件夹里，例如：

```text
src/content/文章/私密相册/index.md
src/content/文章/私密相册/photo-1.jpg
src/content/文章/私密相册/photo-2.jpg
```

本地预览时，需要在项目根目录新建 `.env` 文件。可以复制 `.env.example`，然后改成你的真实密码：

```env
PRIVATE_GALLERY_PASSWORD="你的密码"
```

线上 GitHub Pages 部署时，需要在 GitHub 仓库里添加同名 Secret：

```text
Settings → Secrets and variables → Actions → New repository secret
```

名称填：

```text
PRIVATE_GALLERY_PASSWORD
```

值填你的密码。

注意：

- 这种加密保护的是“文章 HTML 内容”，不是服务器级权限。
- 如果图片路径只写在加密文章里，普通访客看不到图片入口。
- 但仓库如果是公开的，图片原文件仍然在 GitHub 仓库里。因此它适合挡普通人，不适合放真正敏感资料。

## 本地预览和发布

在 PowerShell 进入项目：

```powershell
cd C:\Users\q6058\Documents\Codex\2026-06-24\o-k\outputs\twilight-site
```

本地预览：

```powershell
pnpm dev
```

发布前检查：

```powershell
pnpm build
```

推送到 GitHub：

```powershell
git add .
git commit -m "Update site content"
git push origin main
```

推送成功后，GitHub Actions 会自动部署。通常等 1～3 分钟，网站就会更新。
