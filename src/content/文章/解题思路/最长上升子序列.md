由于数据范围是 $0 \le n \le 1000$，要求的时间复杂度为 $O(n^2)$，空间复杂度为 $O(n)$，最适合采用动态规划来解决。

## 思路

可以把大问题拆解成小问题。

### 1. 定义状态

定义一个一维数组 `dp`，长度与原数组 `arr` 相同。

- `dp[i]` 的含义是：**以 `arr[i]` 结尾的最长严格上升子序列的长度**。

### 2. 状态转移方程

对于每一个位置 `i`，我们要往前遍历所有在它之前的元素 `j`（即 $0 \le j < i$）：

- 如果发现 `arr[i] > arr[j]`，说明 `arr[i]` 可以接在 `arr[j]` 后面，构成一个更长的上升子序列。

- 此时，以 `arr[i]` 结尾的子序列长度就可以更新为 `dp[j] + 1`。

- 因为 `j` 有很多个，要找能拼出的最大长度，所以转移方程为：

    $$dp[i] = \max(dp[i], dp[j] + 1) \quad (\text{条件是 } arr[i] > arr[j])$$

### 3. 初始状态与边界

- 如果 $n = 0$，空数组直接返回 `0`。

- 每个元素自身都可以看作一个长度为 `1` 的子序列，所以 `dp` 数组初始化时每个位置的值都为 `1`。

### 4. 最终答案

整个数组的最长严格上升子序列的长度，就是 `dp` 数组中的**最大值**。

## Go 语言源码实现

```go
package main

import "fmt"

func lengthOfLIS(arr []int) int {
	n := len(arr)
	// 边界条件：如果数组为空，返回 0
	if n == 0 {
		return 0
	}

	// 初始化 dp 数组，每个位置初始长度都为 1
	dp := make([]int, n)
	for i := range dp {
		dp[i] = 1
	}

	maxLen := 1 // 记录全局最大长度

	// 双重循环：i 从前往后遍历，j 遍历 i 之前的元素
	for i := 1; i < n; i++ {
		for j := 0; j < i; j++ {
			// 如果当前元素大于前面的元素，说明可以接在后面构成严格上升子序列
			if arr[i] > arr[j] {
				if dp[j]+1 > dp[i] {
					dp[i] = dp[j] + 1
				}
			}
		}
		// 每次算完一个 dp[i]，更新全局最大值
		if dp[i] > maxLen {
			maxLen = dp[i]
		}
	}

	return maxLen
}

func main() {
	// 测试用例 1
	arr1 := []int{1, 5, 3, 7, 3}
	fmt.Printf("数组 %v 的 LIS 长度为: %d\n", arr1, lengthOfLIS(arr1)) // 输出 3，比如 [1, 3, 7] 或 [1, 5, 7]

	// 测试用例 2
	arr2 := []int{10, 9, 2, 5, 3, 7, 101, 18}
	fmt.Printf("数组 %v 的 LIS 长度为: %d\n", arr2, lengthOfLIS(arr2)) // 输出 4，比如 [2, 3, 7, 101]
}
```

## 复杂度分析

- **时间复杂度**：$O(n^2)$。代码中用了两层嵌套循环。外层循环执行 $n$ 次，内层循环最多执行 $n$ 次，总计算次数约为 $\frac{n^2}{2}$。在 $n \le 1000$ 的情况下，计算量大约是 $5 \times 10^5$ 次。

- **空间复杂度**：$O(n)$。我们创建了一个和原数组等长的 `dp` 切片来存储中间状态，复杂度为 $O(n)$。
